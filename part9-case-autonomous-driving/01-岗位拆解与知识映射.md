# 9.1 岗位拆解：JD 逐条分析与知识映射

## 原始 JD

**工作职责：**

1. 负责无人机、手持等产品的数据平台（Data Platform）的后端架构设计与开发，构建全生命周期管理系统，解决海量非结构化数据的高效存储与 IO 难题。
2. 设计并实现大规模云端调度系统，支持海量虚拟里程测试和数据包处理；负责仿真任务的并发控制、资源隔离、镜像管理及结果自动化评测（Auto-Grading）服务的开发。
3. 开发数据挖掘、自动化标注（Auto-Labeling）及人工标注平台的后端服务；设计灵活的工作流引擎（Workflow），串联"预处理-标注-质检-训练"的自动化生产管线。
4. 算法集成与接口适配：负责仿真器（Simulator）与算法栈（如 Apollo/算法 Docker）之间的通信中间件与接口开发，解决异构系统间的数据转化、时序同步及通信延迟问题。
5. 建设平台的可观测性体系（Monitoring/Logging/Tracing），优化数据处理链路的吞吐量，确保在大规模并发仿真或数据入库场景下的系统高可用性。

**任职要求：**

1. 计算机相关专业本科及以上学历，具备 4 年及以上相关工作经验。
2. 精通 Go / Java / C++ 其中至少一门语言，具备扎实的分布式系统设计理论（CAP/分布式锁/一致性协议）及微服务架构经验；精通 Kubernetes (K8s) / Docker 生态，有 Operator 开发或 K8s 调度器（Scheduler）深度定制经验者优先；理解云资源的弹性伸缩（HPA/VPA）机制。
3. 熟练掌握 MySQL/PostgreSQL 及 NoSQL（MongoDB/HBase），熟悉 Kafka/Pulsar 消息队列；有对象存储（S3/MinIO）的大文件处理经验。
4. 领域相关性：理解自动驾驶或机器人系统的基础架构，熟悉 ROS/CyberRT 通信机制，了解 Rosbag/PCD 等常用数据格式及其解析处理方法。
5. 具备复杂业务抽象能力，能理解"数据闭环"、"仿真"、"场景生成"、"评测指标"、"数据血缘"等业务概念，并将其转化为稳定的技术架构。

## 逐条拆解

### 职责 1：数据平台 + 海量非结构化存储

**JD 关键词：** 数据平台、全生命周期管理、海量非结构化数据、高效存储与 IO

**技术本质：** 自动驾驶采集的数据是多模态、时序性、大文件——LiDAR 点云（PCD 格式，单帧可达 200MB）、摄像头图像（RAW/JPG）、雷达数据、GPS/IMU 航迹。一辆测试车每天产生数 TB 数据，几百辆车就是 PB 级/天。需要解决的是大文件的高吞吐读写、元数据管理、数据生命周期（采集→清洗→标注→训练→归档→销毁）。

### 职责 2：大规模云端调度系统

**JD 关键词：** 云端调度、虚拟里程测试、并发控制、资源隔离、镜像管理、Auto-Grading

**技术本质：** 虚拟里程测试是自动驾驶的核心验证手段——不用物理车在路上跑，而是在仿真环境中跑数百万公里。每个仿真任务是一个独立的 Docker 容器（加载场景地图 + 算法镜像），需要 K8s 调度数百到数千个并发任务。并发控制（任务排队、优先级、资源配额）、资源隔离（CPU/GPU/内存限制）、镜像管理（分层缓存、镜像分发到多节点）、Auto-Grading（仿真结果自动评分：是否碰撞、是否偏离车道、是否超速）。

### 职责 3：工作流引擎 + 标注平台

**JD 关键词：** 数据挖掘、Auto-Labeling、人工标注、工作流引擎、预处理-标注-质检-训练管线

**技术本质：** 这是数据闭环的核心管线。原始数据进来后，先预处理（去重、脱敏、格式转换），然后自动标注（模型推理生成预标注），人工标注员在预标注基础上修正，质检员抽检，最终训练数据入库。工作流引擎需要支持 DAG 任务编排（有依赖关系的任务图）、状态机管理（Pending→Running→Success/Failed→Retry）、断点续跑、优先级调度。

### 职责 4：算法集成 + 通信中间件

**JD 关键词：** Simulator、Apollo、算法 Docker、通信中间件、时序同步、通信延迟

**技术本质：** 仿真器（如 CARLA、LGSVL）模拟传感器输出，算法栈（Apollo CyberRT、自研感知规划算法）接收传感器数据并输出控制指令。通信中间件负责两者之间的数据桥接——把仿真器的虚拟传感器数据按正确的时序喂给算法，把算法的控制指令回传给仿真器。核心挑战是多传感器时序同步（LiDAR 10Hz、Camera 30Hz、Radar 20Hz 需要对齐到同一时间戳）和通信延迟（实时性要求 < 10ms）。

### 职责 5：可观测性体系

**JD 关键词：** Monitoring/Logging/Tracing、吞吐量优化、高可用

**技术本质：** 数千个仿真任务并发跑，每个任务产生日志、指标、调用链。需要统一采集、存储、查询、告警。核心指标：任务吞吐量（tasks/sec）、队列积压深度、GPU 利用率、仿真成功率。核心日志：任务生命周期日志、异常诊断日志。核心追踪：一个仿真任务从提交到完成的完整调用链。
